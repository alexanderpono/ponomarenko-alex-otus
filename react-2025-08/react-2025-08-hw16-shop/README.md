## Описание/Пошаговая инструкция выполнения домашнего задания:
Правила выполнения проекта

В рамках данного учебного проекта у вас есть возможность объединиться в команды по 3 человека для выполнения проектной работы (оптимальное количество, может быть немного изменено в зависимости от количества учащихся).
У вас будет шанс применить все полученные знания на практике. В рамках командного проекта вы научитесь:

    Эффективно работать в группе, распределяя задачи между участниками.
    Разрешать конфликты, которые могут возникнуть в процессе работы с версиями вашего проекта. Обсудим лучшие практики совместной разработки и разрешения конфликтов.
    Объединять различные части проекта, разработанные каждым из членов команды, в единое приложение. Это поможет вам лучше понять, как складываются части проекта в единую систему и как важно выстроить эффективную систему взаимодействия между участниками команды.


В качестве дипломного проекта можно релизовать интернет-магазин, либо приложение учета доходов/расходов

Техническое Задание

    Аутентификация пользователей (регистрация, вход, смена пароля (не обязательно)) + авторизация (экран профиля только для авторизованных пользователей).

## Решение
Реализован функциональный прототип интернет-магазина.
Ссылка в интернете: https://alexanderpono.github.io/ponomarenko-alex-otus/react-2025-08/react-2025-08-hw16-shop/

Исходные тексты проекта: https://github.com/alexanderpono/ponomarenko-alex-otus/tree/react-2025-08-hw16-shop
Сторибук: https://alexanderpono.github.io/ponomarenko-alex-otus/react-2025-08/react-2025-08-hw16-shop/storybook/

1. Реализован выбор языка пользовательского интерфейса
Для этого в шапке сайта выбрать переключатель "Язык/Language", выбрать язык.

2. Реализован выбор темы пользовательского интерфейса
Для этого в шапке сайта выбрать переключатель "Цветовая схема", выбрать название схемы.

3. Реализована регистрация пользователей (используется API бэкенда)
* В правом верхнем углу пользовательского интерфейса нажать на кнопку "Войти". Появится модальное окно "Войти".
* Выбрать "Регистрация"
* Ввести логин, пароль в поля "Пароль, Повторите пароль".

Реализована валидация заполнения обязательных полей "Логин", "Пароль", "Повторите пароль"
Реализовано отображение текста ошибок, если бэкенд вернул ответ с ошибкой.


4. Реализована аутентификация пользователей (используется API бэкенда)
* В правом верхнем углу пользовательского интерфейса нажать на кнопку "Войти". Появится модальное окно "Войти".
* Выбрать "Вход"
* Ввести логин, пароль и нажать на кнопку "Войти".

Реализована валидация заполнения обязательных полей "Логин", "Пароль"
Реализовано отображение текста ошибок, если бэкенд вернул ответ с ошибкой.

Данные аутентификации пользователя сохраняются в localStorage, поэтому нажатие на F5 не влияет на статус авторизации пользователя.


5. Реализовано отображение списка продуктов на главном экране (используется API бэкенда)
Если пользователь неавторизован, то на главном экране отборажается писок продуктов всех участников проекта к курсу "OTUS / React.js"
* в меню разделов доступна часть разделов разделы: Продукты, Корзина

Если пользователь авторизован: 
* на главной странице отображается список продуктов для текущего commandId.
* на карточках продукта отобюражается кнопка "Редактировать"
* на странице вижна кнопка "Добавить новый продукт"
* в меню разделов видны все разделы: Продукты, Категории, Корзина, Профиль

В модальных окнах "Новый продукт" и "Редактирование продукта" доступны поля для редактирования свойств нового или существующего продукта.
Изображение продукта задается внешней ссылкой на изображение: попытка загрузить изображение через API выдает ошибку CORS.

Реализована постраничная подгрузка списка продуктов с использованием API IntersectionObserver.


6. В разделе "Категории" реализовано редактирование / добавление категорий (используется API бэкенда):
* отображается список категорий
* отображаются свойства выделенной категории с возможностью сохранить изменения
* можно нажать на кнопку "Добавить категорию", задать значения полей, сохранить новую категорию.


7. Реализовано управление продуктов в корзине интернет-магазина. Этот функционал доступен независимо от того, авторизован пользователь или нет:
* на главной странице сайта на карточках продуктов доступна кнопка "Добавить в корзину". Эта кнопка видоизменяется, если продукт добавлен в корзину: отображается количество данного продукта в корзине и мини-кнопки "+" и "-", которые позволяют управлять количеством товара в корзине

* в разделе "Корзина" отображается список товаров, добавленных в корзину. По каждому товару отображается изображение, название, описание, количество единиц данного товара в корзине, цена товара, цена всех единиц данного товара. Также есть возможность упправлять количеством данного товара в корзине, есть возможэность удалить товар из корзины.

* в разделе "Корзина" отображается общая стоимость товаров в корзине.


8. Реализован раздел "Профиль пользователя":
* отображается email пользователя и кнопка смены пароля.
* При нажатии на кнопку смены пароля отображается модальное окно с полями для ввода старого и нового паролья. При  нажатии на сохранение - идет запрос к API бэкенда. Если смена прошла успешно - окно закрывается. Если произошла ошибка при смене пароля - в открытом модальном окне смены пароля отображается текст ошибки.
* проводится проверка заполнения обязательных полей в модальном окне.

9. Обработка ошибок серверных запросов
- реализована в модальных окнах Регистрации / Авторизации, Смены пароля

10. Валидация полей
- реализована в модальных окнах Регистрации / Авторизации, Смены пароля, Добавления / Редактирования продукта

### Запуск на локальном компьютере:
1. Клонировать репозиторий

```
git clone git@github.com:alexanderpono/ponomarenko-alex-otus.git
```

2. Перейти в папку с репозиторием
```
cd ponomarenko-alex-otus
```

3. Если ветка react-2025-08-hw16-shop не вмержена в master - переключиться на ветку 
```
git checkout react-2025-08-hw16-shop
```

4. Перейти в папку проекта
```
cd react-2025-08/react-2025-08-hw16-shop
```

5. Установить зависимости (Ожидается версия node.js не ниже 18)
```
npm ci
```

6. Запустить тесты (покрытие неполное)
```
npm run test
```

7. Запустить UI
```
npm run start
```

8. Открыть в браузере адрес http://localhost:2233/
