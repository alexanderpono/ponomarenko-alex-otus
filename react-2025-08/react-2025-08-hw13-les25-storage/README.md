# Домашнее задание 13. Хранилище и защищенный роутинг

Хранилище и защищенный роутинг

## Цель:

научиться создавать глобальные хранилище данных приложения.

## Описание/Пошаговая инструкция выполнения домашнего задания:
Необходимо создать хранилище данных с помощью redux toolkit

В хранилище должен быть

    токен
    инициация приложения (булево значение, срабатывает один раз при загрузке приложения)
    профиль
    корзина (для интернет магазина)
    список товаров/список операций

Токен создается любой при авторизации (необходимо сделать фэйковую авторизацию)

Профиль зависит от токена, если токен есть - добавляем в хранилище фэйковые данные профиля. Если токена нет, очищаем профиль.

Токен должен сохраняться в локальном хранилище и обмениваться между вкладками.

Кнопки "в корзину" в карточке товара добавляют соответствующих товар в корзину. Кнопки "удалить" в карточке товара в корзине, соответственно удаляют товар из корзины.

Также необходимо создать защищенный роутинг, модальное окно добавления/редактирования товара должно быть доступно только админу.

Профиль доступен только авторизованным пользователям.





## Критерии оценки:
    Обмен токеном между вкладками (4 балла):
        (2 балла) Токен сохраняется в локальном хранилище и корректно используется.
        (1 балл) Обновление токена синхронизируется между вкладками.
        (1 балл) Очистка токена также синхронизируется между вкладками.

    Корзина (2 балла):
        (1 балл) Добавление товаров в корзину работает корректно.
        (1 балл) Удаление товаров из корзины работает корректно.

    Хранилище для операций/товаров и профиля (2 балла):
        (1 балл) Операции/товары корректно хранятся в хранилище.
        (1 балл) Профиль корректно создается/очищается в зависимости от наличия токена.

    Добавление/редактирование операций/товаров (1 балл):
        (1 балл) Форма добавления/редактирования операций/товаров работает корректно, данные сохраняются в хранилище.

    Защищенный роутинг (1 балл):
        (1 балл) Реализован защищенный роутинг: модальное окно доступно только админу, а профиль только авторизованным пользователям.





## Решение

1. Реализовано централизованное хранение состояния приложения в хранилище Redux.

2. Рализован класс StorageService, который предоставляет API для хранения данных в LocalStorage браузера.

Токен хранится в LocalStorage, чтобы обеспечить бесшовную работу приложения при перезагрузке страницы F5.
При инициализации приложения токен считывается из LocalStorage контроллером приложения AppController и передается сервисам прикладных API для формирования необходимых заголовков запросов.

Также в LocalStorage хранится логин активного пользователя. При инициализации приложения этот логин копируется в хранилище Redux.

В LocalStorage хранится содержимое корзины, чтобы обеспечить устойчивость приложения к перезагрузке страницы F5. При инициализации приложения корзина копируется в Redux. Обновление корзины идет внутрь Redux и зеркалируется в LocalStorage.

Список товаров загружается из API и помещается в Redux, откуда уже берется для использования компонентами приложения.

3. Токен получается со стороны API при успешной авторизации или регистрации пользователя.

4. Данные профиля очищаются при нажатии на кнопке "Выйти" (из личного кабинета).

5. Кнопки "В корзину", "+", "-" на карточке товара позволяют управлять количеством товара в корзине.

6. Кнопка "редактировать" на карточке товара отображается только для авторизованног опользователя. Модальное окно редактирования товара доступно только авторизованному пользователю.

7. Раздел "Профиль" отображается в меню и доступен только авторизованным пользователям. Попытка вызвать url /profile для неавторизованного пользователя приводит к автоматическому переходу на url /products

8. Раздел "Категории" отображается в меню и доступен только авторизованным пользователям. Попытка вызвать url /categories для неавторизованного пользователя приводит к автоматическому переходу на url /products
