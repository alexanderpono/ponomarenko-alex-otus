# Домашнее задание 12. Роутинг

Роутинг

## Цель:

научиться создавать роутинг в react приложении.

## Описание/Пошаговая инструкция выполнения домашнего задания:
Начиная с этого занятия, на github pages публикуем проект, а не сторибук.

Необходимо создать

    экран профиля. Содержит форму профиля
    экран товаров/операций. Содержит компонент списка товаров и операций
    модальное окно редактирования/создания товара/операции
    экран корзины (для тех, кто делает интернет-магазин). Здесь использовать компонент списка товаров для корзины
    навигацию по экранам в хедере

Защищенный роутинг для соответствующих экранов мы пройдем в последующих уроках


## Критерии оценки:


    Навигация в хэдере (3 балла)
        (2 балла) Навигация в хэдере реализована и функционирует.
        (1 балл) Навигация работает корректно на всех экранах.

    Экран профиля (2 балла)
        (1 балл) Экран профиля создан и включает форму профиля.
        (1 балл) Форма профиля корректно отображается и работает.

    Экран товаров/операций (2 балла)
        (1 балл) Экран товаров/операций создан и включает компонент списка.
        (1 балл) Компонент списка товаров/операций корректно отображает данные.

    Модальное окно редактирования/создания товара/операции (2 балла)
        (1 балл) Модальное окно создано и интегрировано в проект.
        (1 балл) Открытие модального окна зависит от URL, корректно монтируется и размонтируется.

    Экран корзины (1 балл)
        (1 балл) Экран корзины создан (для интернет-магазина) и включает компонент списка товаров для корзины.





## Решение
Приложение доступно по адресу: https://alexanderpono.github.io/ponomarenko-alex-otus/react-2025-08/react-2025-08-hw12-les21-routing/app/
Существует возможность выбрать цветовую схему сайта и язык пользовательского интерфейса сайта.

1. В шапке пользовательского интерфейса присутствует меню разделов. В неавторизованном режиме в меню отображаются разделы "Продукты", "Корзина".
В авторизованном режиме доступны разделы "Продукты", "Категории", "Корзина", "Профиль".
Для входа или регистрации нужно нажать на кнопку "Войти / Выйти" в правом верхнем углу пользовательского интерфейса.

Навигация по разделам реализована при помощи React Hash Router.
При нажатии на пункты меню происходит переключение между разделами. В меню подсвечивается активный раздел.

2. В меню присутствует раздел "Профиль", который отображает email авторизованного пользователя. Можно открыть модальное окно смены пароля текущего пользователя.

3. Страницей по умолчанию является страница Продуктов. Карточки продуктов отображаются с использованием постраничной подгрузки.

4. В авторизованной зоне на карточках продуктов присутствует кнопка "Редактирование", при нажатии на которую отображается модальное окно редактирования данных о продукте.

5. Реализован раздел "Корзина". Если в корзину добавлены элементы, то в разделе "Корзина" отображается соответствующий список продуктов.
