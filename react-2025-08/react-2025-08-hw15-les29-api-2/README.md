# Домашнее задание 15. 

Работа с сервером

## Цель:

научиться взаимодействовать с сервером.

## Описание/Пошаговая инструкция выполнения домашнего задания:
Реализуем настоящую авторизацию, редактирование профиля, редактирование/добавление товаров/операций. Реализуем динамическую подгрузку товаров/операций с сервера.

## Сервера

Сервер для rest api - [https://github.com/React-js-OTUS/otus-rest-server](https://github.com/React-js-OTUS/otus-rest-server) Типы данных, запросы, url смотри в readme.

Сервер для graphql - https://github.com/React-js-OTUS/otus-graphql-server
Смотри readme

Эти же сервера доступны и для создания диплома

## Критерии оценки:
    Авторизация (3 балла)
        (1 балл) Реализована форма авторизации и отправка данных на сервер.
        (1 балл) Корректная обработка успешной авторизации (сохранение токена и перенаправление).
        (1 балл) Обработка ошибок при авторизации (ошибки ввода, неверный логин/пароль).

    Редактирование/добавление товаров/операций (3 балла)
        (1 балл) Реализована форма редактирования/добавления товаров/операций и отправка данных на сервер.
        (1 балл) Корректная отправка данных и обработка успешного ответа от сервера.
        (1 балл) Обработка ошибок при редактировании/добавлении (ошибки ввода, проблемы с сервером).

    Создание заказа (2 балла)
        (1 балл) Реализована форма создания заказа и отправка данных на сервер.
        (1 балл) Корректная отправка данных и обработка успешного ответа от сервера.

    Динамическая подгрузка товаров/операций (2 балла)
        (1 балл) Реализована динамическая подгрузка данных при прокрутке или с помощью IntersectionObserver.
        (1 балл) Корректная отправка запросов на сервер и обновление данных в интерфейсе.

### 1. Регистрация и аутентификация пользователей (используется API бэкенда)
* В правом верхнем углу пользовательского интерфейса нажать на кнопку "Войти". Появится модальное окно "Войти".
* Выбрать "Регистрация"
* Ввести логин, пароль в поля "Пароль, Повторите пароль".

Реализована:

* Валидация обязательных полей (логин, пароль, подтверждение пароля).
* Отображение ошибок, если бэкенд возвращает сообщение об ошибке.

Для аутентификации пользователя (используется API бэкенда):

* В правом верхнем углу пользовательского интерфейса нажать на кнопку "Войти". Появится модальное окно "Войти".
* Выбрать "Вход"
* Ввести логин, пароль и нажать на кнопку "Войти".


Также реализовано:

* Валидация обязательных полей при авторизации.
Отображение ошибок от сервера.
* Сохранение данных аутентификации в localStorage, поэтому * перезагрузка страницы (F5) не приводит к выходу пользователя из системы.



### 2. Отображение и редактирование списка продуктов (используется API бэкенда)

Если пользователь неавторизован, то на главном экране отборажается писок продуктов всех участников проекта к курсу "OTUS / React.js"
* в меню разделов доступна часть разделов разделы: Продукты, Корзина

Если пользователь авторизован: 
* на главной странице отображается список продуктов для текущего commandId.
* на карточках продукта отобюражается кнопка "Редактировать"
* на странице вижна кнопка "Добавить новый продукт"
* в меню разделов видны все разделы: Продукты, Категории, Корзина, Профиль

В модальных окнах "Новый продукт" и "Редактирование продукта" доступны поля для редактирования свойств нового или существующего продукта.
Изображение продукта задается внешней ссылкой на изображение.


### 3. Управление корзиной интернет-магазина реализовано без создания заказа на стороне бэкенда

Функционал управления содержимым корзины доступен независимо от того, авторизован пользователь или нет:
* на главной странице сайта на карточках продуктов доступна кнопка "Добавить в корзину". Эта кнопка видоизменяется, если продукт добавлен в корзину: отображается количество данного продукта в корзине и мини-кнопки "+" и "-", которые позволяют управлять количеством товара в корзине

* в разделе "Корзина" отображается список товаров, добавленных в корзину. По каждому товару отображается изображение, название, описание, количество единиц данного товара в корзине, цена товара, цена всех единиц данного товара. Также есть возможность упправлять количеством данного товара в корзине, есть возможэность удалить товар из корзины.

* в разделе "Корзина" отображается общая стоимость товаров в корзине.


### 4. Постраничная подгрузка списка продуктов

Реализована с использованием IntersectionObserver — подгрузка новых товаров происходит при прокрутке до конца страницы.

### 5. Обработка ошибок серверных запросов
Обработка ошибок реализована в следующих модальных окнах:

* Регистрация / Авторизация
* Смена пароля

При возникновении ошибки отображается понятное сообщение пользователю.

### 6. Валидация полей формы
Валидация реализована во всех формах:

* Регистрация / Авторизация
* Смена пароля
* Добавление и редактирование продукта

Включает проверку обязательных полей, совпадения паролей, корректности ввода и т.д.
