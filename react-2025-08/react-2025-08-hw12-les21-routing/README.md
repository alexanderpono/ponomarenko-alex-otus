# Домашнее задание 12. Роутинг

Роутинг

## Цель:

научиться создавать роутинг в react приложении.

## Описание/Пошаговая инструкция выполнения домашнего задания:
Начиная с этого занятия, на github pages публикуем проект, а не сторибук.

Необходимо создать

    экран профиля. Содержит форму профиля
    экран товаров/операций. Содержит компонент списка товаров и операций
    модальное окно редактирования/создания товара/операции
    экран корзины (для тех, кто делает интернет-магазин). Здесь использовать компонент списка товаров для корзины
    навигацию по экранам в хедере

Защищенный роутинг для соответствующих экранов мы пройдем в последующих уроках


## Критерии оценки:


    Навигация в хэдере (3 балла)
        (2 балла) Навигация в хэдере реализована и функционирует.
        (1 балл) Навигация работает корректно на всех экранах.

    Экран профиля (2 балла)
        (1 балл) Экран профиля создан и включает форму профиля.
        (1 балл) Форма профиля корректно отображается и работает.

    Экран товаров/операций (2 балла)
        (1 балл) Экран товаров/операций создан и включает компонент списка.
        (1 балл) Компонент списка товаров/операций корректно отображает данные.

    Модальное окно редактирования/создания товара/операции (2 балла)
        (1 балл) Модальное окно создано и интегрировано в проект.
        (1 балл) Открытие модального окна зависит от URL, корректно монтируется и размонтируется.

    Экран корзины (1 балл)
        (1 балл) Экран корзины создан (для интернет-магазина) и включает компонент списка товаров для корзины.
