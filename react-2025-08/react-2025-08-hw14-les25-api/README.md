# Домашнее задание 13. Хранилище и защищенный роутинг

Работа с сервером, основы

## Цель:

научиться работать с сервером, разобраться с разными способами работы с сервером.

## Описание/Пошаговая инструкция выполнения домашнего задания:
Задача - реализовать регистрацию нового пользователя. Используйте наш боевой rest api сервер

Необходимо создать компонент регистрации нового пользователя. Компонент представляет собой форму регистрации, содержит инпут ввода email, password и кнопку отправки данных. Компонент должен обрабатывать ошибки, которые могут приходить в сервера, в частности ошибку о том, что email не валиден. Валидации на стороне фронтенда реализовывать не надо, чтобы проверяющий мог оценить обработку ошибок.

Существует 4 возможных варианта реализации:

    отправлять запрос в функциональном компоненте;
    использовать redux-toolkit-query;
    использовать redux-thunk;
    использовать redux-saga;

Для успешного выполнения задания необходимо использовать как минимум 2 варианта. На каждый вариант нужно написать свой компонент. При желании, вы можете реализовать все 4 варианта.


## Критерии оценки:


    Реализация компонентов (6 баллов):
        (3 балла) Реализация компонента с отправкой запроса в функциональном компоненте.
        (3 балла) Реализация компонента с использованием redux-toolkit-query.
        (3 балла) Реализация компонента с использованием redux-thunk.
        (3 балла) Реализация компонента с использованием redux-saga.

    Максимум за 2 варианта реализаций: (3 + 3) = 6 баллов.
    Каждый правильно реализованный вариант получает 3 балла. Если реализованы все 4 варианта, учитываются первые два.

    Обработка ошибок (4 балла):
        (2 балла) Корректная обработка ошибок от сервера (общая для всех вариантов).
        (2 балла) Специфическая обработка ошибок (например, неверный email).
