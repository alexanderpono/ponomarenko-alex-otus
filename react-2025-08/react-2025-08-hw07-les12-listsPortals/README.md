# Домашнее задание 6. Добавляем состояния в наше приложение

## Цель:

освоить работу с состоянием приложения.

## Описание/Пошаговая инструкция выполнения домашнего задания:

Задание состоит из 3 обязательных пунктов.

    Управление состоянием модального окна.
    Переключение цветовой темы
    Переключение языка

### Состояние модального окна

Нужно создать дополнительный компонент в сторибук. В нем будет кнопка, при нажатии на которую будет открываться модальное окно, а также чуть выше кнопки - текстовый инпут. Текст из этого инпута должен отображаться в модальном окне.

### Переключение цветовой схемы

Нужно написать Провайдер, который с помощью контекста будет переключать цветовую схему во всем проекте. Если вы используете sass, можете подсмотреть решение в учебном проекте. Если используете css-in-js, можно самостоятельно попробовать найти подходящее решение.

Провайдер используется в компоненте App

Также необходимо реализовать переключатель цветовой схемы отдельным компонентом и посместить его в Header.

Переключатель цветовой схемы также нужно реализовать в StoryBook.

### Переключение языка

Аналогично, нужно написать Провайдер, который с помощью контекста будет переключать язык во всем проекте. Используйте react-i18next, можете подсмотреть решение в учебном проекте. Да, мы это не проходили, но все довольно просто. Если будут вопросы - пишите в чат.

Переключение языка нужно реализовать именно через провайдер, для тренировки и закрепления навыков работы с контекстом (такие кейсы также иногда встречаются на реальных проектах, и необязательно с использованием i18n, поэтому в целом работа с переключением языка приложения через контекст будет полезна.

Провайдер стоит использовать в компоненте App

Также необходимо реализовать переключатель языка отдельным компонентом и посместить его в Header.

Переключатель языка также нужно реализовать в StoryBook, вместе с любым компонентом, который позволит увидеть, что язык действительно переключается.


## Критерии оценки:
    Управление состоянием модального окна (3 балла):
        (1 балл) Компонент, содержащий инпут и кнопку, создан и работает корректно.
        (1 балл) При нажатии на кнопку открывается модальное окно.
        (1 балл) Текст из инпута отображается в модальном окне.

    Переключение цветовой схемы (4 балла):
        (1 балл) Проект использует провайдер для управления цветовой схемой.
        (1 балл) Провайдер используется в компоненте App.
        (1 балл) Переключатель цветовой схемы реализован как отдельный компонент и помещен в Header.
        (1 балл) Переключатель цветовой схемы добавлен в сторибук и корректно работает.

    Переключение языка (3 балла):
        (1 балл) Проект использует провайдер для управления языком.
        (1 балл) Провайдер используется в компоненте App.
        (1 балл) Переключатель языка добавлен в сторибук и корректно работает, язык действительно меняется.

Максимальное количество: 10 баллов
Статус "Принято" ставится при наборе 7 и более баллов

    Принципы программирования
        - Знать и уметь использовать react паттерны в соотвествии с принципами программирования
    State Management (Redux)
        - Создавать и настраивать хранилище
